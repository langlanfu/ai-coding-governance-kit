# Secure Code Review Checklist for AI-Assisted Changes

Use this checklist for pull requests where AI tools contributed code, tests, configuration, or documentation.

## General Review

- [ ] The PR describes where AI was used and what was manually verified.
- [ ] The diff is small enough for meaningful review.
- [ ] The reviewer can explain the changed behavior.
- [ ] Generated code does not include unexplained copied snippets or suspicious licenses.

## Secrets and Data

- [ ] No API keys, tokens, passwords, private keys, credentials, or customer data appear in code, tests, fixtures, prompts, or logs.
- [ ] Example data is synthetic.
- [ ] Error messages and logs do not expose sensitive values.

## Authentication and Authorization

- [ ] Auth checks are explicit and tested.
- [ ] No AI-generated bypass, debug role, hardcoded user, or permissive fallback exists.
- [ ] Access control is enforced server-side.

## Input Handling

- [ ] Inputs are validated and normalized.
- [ ] SQL, shell, template, path, and prompt injection risks are considered.
- [ ] File upload and parsing paths handle malicious content safely.

## Dependencies and Supply Chain

- [ ] New dependencies are necessary, maintained, and pinned appropriately.
- [ ] Install scripts and transitive risk are reviewed.
- [ ] License compatibility is checked for business use.

## Tests

- [ ] Tests cover expected and failure paths.
- [ ] Tests were not weakened to pass.
- [ ] Security-sensitive changes include regression tests.
- [ ] Manual verification steps are documented.

## Operations

- [ ] Database migrations include backup and rollback notes.
- [ ] CI/CD or infrastructure changes require owner review.
- [ ] Production impact is understood and approved.
