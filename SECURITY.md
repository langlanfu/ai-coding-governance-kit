# Security Policy

## Scope

This repository contains governance documentation and templates. It does not contain production software, hosted services, or security controls that can certify another project.

Security-relevant reports may include:

- Accidental inclusion of real secrets or credentials in repository content.
- Unsafe guidance that encourages exposing secrets, bypassing review, or granting excessive agent permissions.
- Broken examples that materially weaken the intended governance posture.

Out of scope:

- Requests for legal certification or compliance approval.
- Vulnerabilities in third-party AI tools referenced by the documentation.
- Claims that require access to private systems not part of this repository.

## Reporting

If this project is published, report security concerns using the repository's configured private vulnerability reporting process or contact method. Until then, report issues to the local project owner.

## Handling Expectations

- Preserve evidence without spreading secrets.
- Rotate any exposed credential immediately.
- Remove or redact sensitive content from documentation.
- Record the fix in the changelog when appropriate.

## Disclaimer

This kit is a governance aid, not legal advice, a security certification, or a guarantee that AI-assisted development is safe.
